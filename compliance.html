CSSOM and CSP Compatibility

ZapCaptcha uses CSSOM-injected rules to maintain a strict Content Security Policy (CSP) without inline styles or scripts. All CSS is dynamically injected into a same-origin style sheet and all resources can be served with permissive Access-Control-Allow-Origin headers from zapcaptcha.com. No external CSS fetches or CORS-dependent operations are performed, so no external API calls, XHR calls, and no connect-src directives are needed. This improves XSS resilience and supports enterprise-grade CSP configurations. All animations and layout rules are declared securely via JavaScript-injected stylesheets.
